EtherIP / L2TPv3 over IPsec Client Setting

The following Virtual Hub connection settings will be applied only when EtherIP / L2TPv3 over IPsec Client is attempting to connect this VPN Server with ISAKMP (IKE) Phase ID which is exactly equal to the value specified below.

The ID must be exactly same as a ID in the configuration of the EtherIP / L2TPv3 Client. You can specify IP address as well as characters as ID, if the EtherIP Client uses IP address as Phase 1 ID. If you specify '*' (asterisk), it will be a wildcard to match any clients which doesn't match other explicit rules.

Note that the username and the password must be same as a registered on the Virtual Hub. An EtherIP / L2TPv3 Client will be regarded as it is connected to the Virtual Hub with the identification of the above user information.
